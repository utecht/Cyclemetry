# Install on Linux

<Callout type="info">
  Linux packages are distributed as release artifacts. Download the package that matches your architecture.
</Callout>

## Install

1. Download the Linux artifact from [GitHub Releases](https://github.com/walkersutton/cyclemetry/releases).
2. Unzip the artifact.
3. Install the Debian package:

```sh
sudo apt install ./deb/cyclemetry_*_arm64.deb
```

Use the `.deb` file that matches your system architecture if the artifact includes multiple packages.

## Build From Source

If you prefer to build Cyclemetry yourself, it is a [Tauri](https://tauri.app/) app and can be built on Linux with the Tauri prerequisites installed.

### Prerequisites

Install the [Tauri Linux dependencies](https://tauri.app/start/prerequisites/#linux) for your distribution. For Ubuntu/Debian:

```sh
sudo apt update
sudo apt install libwebkit2gtk-4.1-dev \
  build-essential \
  curl \
  wget \
  file \
  libxdo-dev \
  libssl-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev
```

You'll also need:

- [Node.js](https://nodejs.org/) (v20+)
- [pnpm](https://pnpm.io/)
- [Rust](https://www.rust-lang.org/) via `rustup`
- [FFmpeg](https://ffmpeg.org/) on your `PATH`

### Build

```sh
git clone https://github.com/walkersutton/cyclemetry.git
cd cyclemetry
pnpm install
pnpm build
```

The built AppImage or `.deb` will appear in `src-tauri/target/release/bundle/`.

## Ubuntu Troubleshooting

### "Apple ProRes decoder" codecs are required to play this video

If Ubuntu's built-in Video Player cannot preview a `.mov` generated by Cyclemetry and asks for Apple ProRes decoder codecs, install the restricted media packages:

```sh
sudo apt install ubuntu-restricted-extras libavcodec-extra
```

After installing those packages, the generated `.mov` should preview in the built-in Video Player application.
